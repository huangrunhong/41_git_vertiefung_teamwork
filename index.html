Heute lernen wir Git und Github etwas besser kennen

Was fur Schritte kennen wir bereits?

  1. git init ===>erstellt ein lokales git repo
  2. Wie bereiten wir Dateien vor?
    • git add . ===>bereitet alle Dateien Anderungen fur die Staging Area vor
    • git add "Datei.name" ===>bereitet eine einzelne Datei/Anderung fur die Staging Area vor
  3. git commit -m "message oder hier bitte etwas anssagekraftiges" ===>speichert die staging area im lokalen git repo und erstellt einen sogenannten "snapshot"
  4. git remote add origin "git@github.com:LinkZumGitHubRepo" ===> erstellen eine Verbindung zwischen dem lokalen git repo auf github
  5. git branch -M main ==> erstellt uns einen main branch auf dem wir dann arbeiten
  6. git push -u origin main ==> ladt lokales repo in das remote repo hoch, in den main branch

!!!Schritte 4 bis 6 bekommt ihr von GitHub nach erstellen eines Repositories!!!

Was ist ein Branch ?

Ein Branch ist ein seperates Arbeitsverzeichnis in unserem Respository, es konnen mehrere branches in einem Repo existieren und unabhangig voneinander bearbeitet werden.

was ist eine Repository ?

Respository = Depot/Lager, hier werden alle von uns hinzugefugen Dateien gespeichert, um sie spater gebundelt an einem Platz zu haben.

Was ist die Stage/Staging area ?

ist wie ein wartezimmer beim Arzt, hier werden die von uns geandert Dateien vorbereitet um ins Repo zugelangen.

Weitere Git- Befehle :

  • git log ===> zeigt uns daten uber unsere commits snapshots an
  • git branch ===> zeigt uns alle verfugbaren branches an und auf welchem wir uns gerade befinden
  • git checkout "name"===> lasst uns zwischen unseren lokalen branches wechseln
